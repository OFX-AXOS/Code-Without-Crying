HTML Syntax

Understanding the basic structure and rules of writing HTML code. Learn how elements work, what tags are, and how to write valid HTML.

8 min read Beginner HTML
Part 1

What is HTML Syntax?

HTML syntax refers to the rules and structure for writing HTML code. Every HTML document follows a specific pattern that browsers understand. Think of it like grammar rules for a language, but for web pages.

Key Point

HTML uses tags to define elements. Tags are wrapped in angle brackets like <tag>. Most tags come in pairs: an opening tag and a closing tag.

Example 
<p>This is a paragraph.</p>
<h1>This is a heading.</h1>
<a href="page.html">This is a link.</a>

Notice how each element has an opening tag and a closing tag. The closing tag includes a forward slash / before the tag name.

Part 2

Anatomy of an HTML Element

1

Opening Tag

The opening tag marks the beginning of an element. Example: <div>

2

Content

The content goes between the opening and closing tags. This is what users see on the page.

3

Closing Tag

The closing tag ends the element. It looks like the opening tag but with a forward slash: </div>

Common Mistake

Forgetting to close tags is a common error. Always make sure every opening tag has a matching closing tag, except for self-closing tags like <img /> or <br />.

Nested Elements

Elements can be placed inside other elements. This is called nesting.

Attributes

Attributes provide additional information about elements, like href in links.

Self-Closing Tags

Some tags don't need a closing tag, like <img /> and <br />.

Part 3

HTML Attributes

Attributes provide extra information about HTML elements. They're always specified in the opening tag and usually come in name/value pairs.

HTML 
<a href="https://example.com" target="_blank">
  Visit Example
</a>

<img src="photo.jpg" alt="Description" width="500" />

<div class="container" id="main">
  Content here
</div>

Remember This

Attributes always follow the pattern: attribute="value". The value is wrapped in quotes.
