HTML Introduction (a.k.a. Talking Nicely to Browsers)

HTML is not hacking. It is not magic. It is you describing content to a browser and hoping it behaves. Sometimes it does.

~7 min (no refunds) Beginner HTML
Part 1

HTML Tags: Tiny Brackets, Serious Consequences

HTML stands for HyperText Markup Language. It sounds dramatic. In practice, it just labels things so browsers do not guess and embarrass themselves.

HTML Tag Lore

Tags look like <p>. Most of them need a matching </p>. Forget it, and the browser will invent structure. Poorly.

HTML 
<h1>Hello World</h1>
<p>This is HTML. I am learning.</p>

VS Code Cheat Code

Create a file named index.html. Type ! and press Enter. VS Code generates the boilerplate because typing it manually is pointless.

Skill Issue Alert

If nothing happened, Emmet is disabled. Or you pressed the wrong key. Browsers do not respond to hope.

Below is what the browser actually sees: a nested structure. Not magic. Just boxes inside boxes.

<html>
<head>
<title>Page title</title>
</head>
<body>
<h1>This is a heading</h1>
<p>This is a paragraph.</p>
<p>This is another paragraph.</p>
</body>
</html>

What You Learned (Whether You Tried or Not)

  • HTML labels content. That is its entire job.
  • Tags usually come in pairs.
  • The basic structure is mandatory.
  • Emmet exists to save time, not judge you.
